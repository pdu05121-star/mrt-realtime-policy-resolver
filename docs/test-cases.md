# 테스트 케이스 (검증용 샘플 문의)

Codex에 플러그인을 설치한 뒤, 아래 세 문의를 그대로 입력해 스킬이 트리거되고
SKILL.md의 4단계 절차(분류 → 조회 → 계산 → 출력)를 따르는지 확인합니다.

## 케이스 1 — 항공권 일정변경
**입력**: "다음 주 화요일 출발 항공권인데, 하루 앞당기고 싶어요. 지금 바로 가능한지, 수수료는 얼마나 되는지 알려주세요."

**기대 동작**: 상품 유형=항공권, 요청=일정변경으로 분류 → 마이리얼트립 항공권 변경 정책 페이지 + 해당 항공사 공개 변경 수수료 규정 조회 → 옵션(가능/수수료 조건)과 예상 처리 시간을 출처와 함께 출력.

## 케이스 2 — 투어 취소·대체 (긴급)
**입력**: "내일 세부 출발인데 스노클링 투어가 갑자기 이번달 전체 운영 안 한다고 해요. 다른 비슷한 액티비티로 바꿀 수 있나요?"

**기대 동작**: 긴급도=최우선으로 표시 → 세부 지역 유사 액티비티 공개 목록 조회 → 대체 가능 옵션 + 기존 상품 환불 조건을 우선순위 상단에 배치.

## 케이스 3 — 예약 오류(확정 후 취소 통보)
**입력**: "분명 예약 확정 메일을 받았는데, 다시 보니 취소됐다고 해요. 이거 어떻게 해야 하나요?"

**기대 동작**: 확인 불가 항목(공급사 측 실시간 재고 상태)을 명시적으로 분리하고,
확정적으로 답할 수 없는 부분에 대해서는 "상담원 확인이 필요합니다"로 표기하는지 확인.
(추측성 답변을 만들어내지 않는지가 이 케이스의 핵심 검증 포인트)

## 검증 시 확인할 것
- [ ] 모든 옵션·소요시간에 출처 URL이 붙어 있는가
- [ ] 공개 자료로 확인 안 되는 부분을 "확인 불가"로 분리했는가 (임의로 지어내지 않았는가)
- [ ] 긴급 문의가 우선순위 상단에 배치되는가
- [ ] 출력 형식이 SKILL.md에 정의된 5개 섹션(문의 요약/옵션/처리시간/확인필요/출처)을 따르는가
