# 실시간 정책 통합 조회 (Realtime Policy Resolver)

## 왜 필요한가

여행 상품(항공권/투어·액티비티/숙소)의 일정변경·취소·오류 문의는 상품마다 답이 다른 곳에 있습니다.

- 항공권 변경/환불 → 항공사 약관 + 마이리얼트립 자체 정책
- 투어·액티비티 취소 → 현지 공급사(파트너) 정책 + 계절/재고 상황
- 숙소 변경 → 숙소 자체 취소 규정 + 플랫폼 규정

문제는 이 정보가 한 곳에 없다는 것입니다. 그래서 상담원도 고객도 "확인 후 안내드리겠습니다" → 1~3영업일 대기라는 패턴이 반복됩니다. 이 스킬은 그 통합 조회를, 사람이 여러 문서를 오가며 하는 대신 자동으로 수행합니다.

## 언제 이 스킬을 사용하는가

다음과 같은 문의가 들어왔을 때 트리거합니다:

- "항공권 일정을 바꾸고 싶어요"
- "이 투어 상품 취소하면 얼마나 환불되나요"
- "예약이 확정됐는데 실제로는 안 된다고 해요, 어떻게 해야 하나요"
- 그 밖에 "지금 당장 답이 필요한" 여행 상품 변경/취소/오류 문의

## 진행 절차

### 1단계 — 문의 분류
문의 내용에서 다음을 먼저 식별합니다:
- 상품 유형: 항공권 / 투어·액티비티 / 숙소 / 기타
- 요청 유형: 일정변경 / 취소·환불 / 예약 오류(확정되었으나 이용 불가) / 단순 문의
- 긴급도: 출발까지 남은 시간 (예: "내일 출발"이면 최우선 처리 대상으로 표시)

### 2단계 — 공개 정책 조회
상품 유형에 맞는 **공개된** 정책 문서를 조회합니다. 절대로 내부 시스템 접근을 가정하지 않고, 다음과 같이 실제로 웹에서 확인 가능한 자료만 근거로 사용합니다:

- 마이리얼트립 고객센터/도움말 페이지 (운영시간, 환불·변경 규정, FAQ)
- 항공권의 경우: 해당 항공사의 공개 변경/환불 수수료 규정
- 투어·액티비티의 경우: 공급사가 공개한 취소 규정, 계절성 운영 정보 (예: "우기에는 스노클링 미운영" 같은 공지)
- 숙소의 경우: 해당 숙소/플랫폼의 공개 취소 정책

각 자료는 출처(URL)를 함께 기록합니다. 출처가 불명확하거나 비공개 정보로만 뒷받침되는 항목은 "확인 불가"로 명시하고, 임의로 답을 만들지 않습니다.

### 3단계 — 옵션 계산
조회한 정책을 조합하여 다음을 산출합니다:
- 지금 시점 기준으로 **실제로 가능한 옵션 목록** (예: "3일 이내 무료 변경 가능" / "수수료 X% 발생" / "정책상 불가, 대안은 Y")
- 각 옵션별 **예상 처리 시간** (공개된 SLA·운영시간 기준)
- 정책만으로 판단이 안 되는 부분은 "이 부분은 상담원 확인이 필요합니다"로 명확히 분리

### 4단계 — 출력
아래 형식으로 정리합니다.

```markdown
## 문의 요약
- 상품 유형: {유형}
- 요청: {요청 내용}
- 긴급도: {긴급도}

## 지금 가능한 옵션
1. {옵션명} — {조건} (근거: {출처 URL})
2. {옵션명} — {조건} (근거: {출처 URL})

## 예상 처리 시간
- {옵션}: {소요 시간} (근거: {출처 URL})

## 상담원 확인이 필요한 부분
- {항목}: {이유}

## 참고 출처
- {URL 1}
- {URL 2}
```

## 지켜야 할 원칙

- **추측 금지**: 공개 자료로 확인되지 않는 내용은 절대로 "아마 가능할 것"처럼 서술하지 않습니다. 확인 불가 항목은 명시적으로 분리합니다.
- **출처 필수**: 모든 옵션과 처리 시간에는 근거 URL을 함께 표기합니다.
- **최신성 확인**: 정책 페이지의 게시일/업데이트 시점을 확인하고, 오래된 정보일 경우 그 사실을 함께 알립니다.
- **긴급 문의 우선순위화**: 출발이 임박한 문의는 "즉시 조치 가능한 옵션"을 맨 위에 배치합니다.

## 예시

**입력**: "내일 세부 출발인데 스노클링 투어가 갑자기 이번달 전체 운영 안 한다고 해요. 다른 비슷한 액티비티로 바꿀 수 있나요?"

**처리**:
1. 분류 → 투어·액티비티 / 일정변경(대체 상품) / 긴급(익일 출발)
2. 조회 → 마이리얼트립 세부 지역 액티비티 목록 공개 페이지, 해당 공급사 운영 공지, 환불 정책 페이지
3. 계산 → 대체 가능한 유사 액티비티 목록, 각각의 예약 가능 여부와 환불 정책 차이
4. 출력 → 위 형식에 맞춰 "지금 예약 가능한 대체 상품 2건 + 기존 상품 환불 조건 + 상담원 확인 필요 항목(공급사 재고 실시간 반영 여부)"
