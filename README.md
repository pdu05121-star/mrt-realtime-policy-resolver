# 마이리얼트립 실시간 정책 통합 조회 플러그인

## 1. 무엇을, 누가, 어떤 상황에서 쓰나요?

마이리얼트립 CS 상담원 또는 고객 자신이, 항공권 일정변경·투어 취소·예약 오류 등
**지금 당장 답이 필요한 여행 상품 문의**를 처리할 때 사용합니다. 문의를 입력하면
상품 유형(항공권/투어·액티비티/숙소)에 따라 서로 다른 곳에 흩어져 있는 공개 정책·
운영시간·수수료 정보를 실시간으로 교차 조회하여, "확인 후 안내드리겠습니다" 대신
지금 가능한 구체적 옵션과 예상 처리 시간을 즉시 산출합니다.

## 2. 왜 이 문제를 선택했나요?

공개된 이용후기·커뮤니티 글을 조사한 결과, 마이리얼트립의 CS 문제는 "채널이 없다"가
아니라 "채널은 있지만 실시간 통합 답변이 안 된다"는 데 있었습니다.

- 항공 일정변경 문의 시 1:1 채팅에는 안내만 있고 구체적 날짜 선택 화면이 없으며,
  처리에 1~3영업일이 걸린다는 이용자 후기가 확인됩니다.
- 전화 상담은 09:00~18:00(평일 기준 항공권 환불/변경은 09:00~17:00)로 운영시간이
  제한적이고, 파트너 지원센터 역시 평일 09:00~18:00만 운영됩니다.
- 예약이 확정 표시되었으나 실제로는 공급사 사정으로 불가능했던 사례(스노클링 투어
  전체 미운영 통보)에서도, 대체 옵션을 즉시 안내받지 못하고 연락이 두절되는 경험이
  반복적으로 보고됩니다.

단순 CS 챗봇으로 보일 수 있지만, 실제로는 항공사/현지 공급사/숙소마다 제각각인
정책과 재고 데이터가 파편화되어 있어 통합 자체가 구조적으로 어려운 문제라고
판단했습니다. 이는 저희 팀이 여행 서비스(KSPOT)를 만들며 겪은 "여러 외부 데이터
소스를 하나의 판단으로 통합하는 어려움"과 본질적으로 같은 문제이며, 그 경험을
일반화해 이 스킬을 설계했습니다.

## 3. 플러그인은 어떻게 작동하나요?

`src/skills/realtime-policy-resolver/SKILL.md`에 정의된 절차를 따릅니다.

1. **문의 분류**: 상품 유형, 요청 유형(일정변경/취소/오류), 긴급도(출발까지 남은 시간)를 식별
2. **공개 정책 조회**: 마이리얼트립 고객센터 페이지, 항공사 공개 변경 규정, 공급사 공지, 숙소 취소 정책 등 공개 자료만 조회 (내부 시스템 접근 가정 없음)
3. **옵션 계산**: 조회한 정책을 조합해 지금 가능한 옵션, 예상 처리 시간, 확인 불가 항목을 분리해 산출
4. **정형 출력**: 문의 요약 / 지금 가능한 옵션 / 예상 처리 시간 / 상담원 확인 필요 항목 / 참고 출처 순으로 출력

모든 항목에 출처 URL을 필수로 표기하며, 공개 자료로 확인되지 않는 내용은
"확인 불가"로 명시하고 추측하지 않습니다.

## 4. AI를 어떻게 활용했나요?

- Claude(Anthropic)를 사용해 (1) 마이리얼트립 관련 공개 후기·뉴스·고객센터 문서를
  웹 검색으로 조사하고, (2) 조사 결과를 바탕으로 실제 문제(CS 실시간성 부재의 구조적
  원인)를 좁혀나가는 대화를 진행했습니다.
- 초기에는 "CS 자동 응대 봇"으로 문제를 단순하게 정의했으나, "기업이 안 만들었다면
  이유가 있을 것"이라는 검토 과정을 거쳐 "정책 파편화로 인한 통합 조회의 어려움"으로
  문제 정의를 재구성했습니다. 이 재정의 과정 자체가 AI와의 대화를 통해 이루어졌습니다.
- SKILL.md의 절차 설계(4단계 프로세스, 출력 형식, 원칙)는 Claude와 함께 초안을
  작성한 뒤, 팀이 실제 KSPOT 개발 과정에서 사용해온 "판단 원칙"(추측 금지, 출처 필수,
  확인 불가 항목 분리)을 반영해 다듬었습니다.
- 전체 대화 로그는 `logs/` 폴더에 원본 그대로 포함되어 있습니다.

## 5. 어떻게 검증했나요?

- 문제 정의에 사용한 모든 근거(CS 운영시간, 처리 소요시간, 이용자 후기)는 각각
  공개 URL로 확인 가능하며, 대화 로그와 아래 출처 목록에 남겨두었습니다.
- 스킬 자체는 "확인 불가 항목을 명시적으로 분리"하는 원칙을 두어, 검증되지 않은
  정보를 확정적으로 제시하지 않도록 설계했습니다.
- (실행 검증) Codex CLI에 플러그인을 설치한 뒤 `docs/test-cases.md`에 정리된
  샘플 문의 3건(항공권 일정변경 / 투어 취소·대체 / 예약 오류)을 입력해 출력 형식과
  출처 표기가 절차대로 나오는지 확인했습니다.

## 설치 및 실행 방법 (Codex CLI 기준)

1. `src` 폴더를 로컬 플러그인 경로로 복사하거나, 프로젝트 루트에 `.agents/plugins/marketplace.json`을 추가해 `source.path`가 `src`를 가리키도록 설정합니다.
2. Codex를 재시작한 뒤 `/plugins`에서 `myrealtrip-realtime-policy-resolver`가 보이는지 확인합니다.
3. 프롬프트에서 예시 문의(예: "내일 세부 출발인데 스노클링 투어가 갑자기 이번달 전체 운영 안 한다고 해요")를 입력해 스킬이 트리거되는지 확인합니다.
